# threads 모듈 문서

이 문서는 `src/threads/` 모듈을 다음 작업자가 바로 이어받을 수 있게 설명한다.
소스 코드를 먼저 읽고, 파일 경계와 함수 경계를 기준으로 책임을 적는다.
여기서 말하는 동기화는 코드 변경 뒤 이 문서와 테스트 관점도 함께 맞추는 일을 뜻한다.
외부 웹 검색 없이 로컬 소스와 포팅 계획 문서만 근거로 삼는다.

## Module Responsibility

`src/threads/`는 Threads 계정의 최신 글을 로그인 없는 공개 경로로 가져오려는 모듈이다.
먼저 Threads 페이지에서 LSD token을 찾고 Instagram profile info에서 user id를 얻은 뒤, 여러 GraphQL doc_id를 순차 시도한다.
차단이나 doc_id 실패가 잦은 플랫폼 특성 때문에 빈 결과와 프론트 fallback이 정상 상태로 취급된다.

### 책임 경계

- Threads 계정 source 등록을 소유한다.
- LSD token과 Instagram user id 조합을 소유한다.
- GraphQL POST 요청과 doc_id fallback loop를 소유한다.
- 중첩 post object 파싱을 소유한다.

### 운영 관점

- Post item은 `account`, `text`, `likes`, `replies`, `reposts`, `views`, `media`, `url`, `createdAt`를 가진다.
- views는 현재 0으로 고정된다.
- 프론트는 빈 posts를 실패가 아니라 fallback 링크 상태로 보여준다.

## Key Function Signatures

아래 함수명은 실제 소스의 공개 함수와 내부 헬퍼를 기준으로 한다.
테스트 헬퍼는 동작 계약을 보여주는 경우에만 포함한다.

### `register()`

- `threads` source를 `threads_accounts.json`에 연결한다.

### `_threads_lsd_and_userid(username: str)`

- Threads profile HTML에서 LSD token을 찾는다.
- Instagram profile info API에서 user id를 찾는다.
- 둘 중 실패해도 예외 대신 `(lsd, user_id)`를 반환한다.

### `fetch_threads_posts(username: str)`

- LSD와 user_id가 없으면 빈 리스트를 반환한다.
- THREADS_DOC_IDS를 순회하며 GraphQL 요청을 보낸다.
- 파싱된 posts가 나오면 즉시 반환한다.

### `_parse_threads(data, username)`

- 중첩 dict/list를 재귀 순회한다.
- caption이 있는 post dict를 프론트 post shape로 변환한다.

### `get_threads_posts(force: bool)`

- 계정 목록을 읽고 최대 5 worker로 fetch한다.
- 캐시 key는 `("threads", tuple(accounts))`이다.
- 반환값은 `(posts, accounts, fetched_at)`이다.

### 테스트 함수 지도

- `ThreadsToolTest.test_parse_threads_nested_post`
- `ThreadsToolTest.test_threads_lsd_and_userid_uses_ig_app_id`

## Dependents

이 모듈을 import하거나 HTTP 라우트로 소비하는 쪽이다.

- `src/threads/__init__.py` re-exports `get_threads_posts`, `register`
- `src/main.py` calls `threads_tool.register()` at import time
- `src/main.py` calls `threads_tool.get_threads_posts` in `/api/threads`
- `src/main.py` account POST route updates `threads` source
- `src/frontend/index.html` calls `/api/threads` and `/api/threads/accounts`
- `src/frontend/index.html` shows `showThreadsFallback` when posts are empty
