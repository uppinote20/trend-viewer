## Module Responsibility

`src/tiktok/`는 tikwm API를 통해 한국 지역 인기 피드와 등록 계정 영상을 함께 모은다.
트렌딩과 계정별 posts를 합친 뒤 video id 기준으로 중복을 제거한다.
프론트는 이 데이터를 쇼츠형 카드 그리드로 표시하고 조회수·좋아요·댓글순 정렬을 수행한다.

### 책임 경계

- tikwm endpoint 상수를 소유한다.
- TikTok region 정책을 소유한다.
- video item 정규화 헬퍼를 소유한다.
- 트렌딩과 계정 fetch 병합 및 dedupe를 소유한다.

### 운영 관점

- TikTok item은 `account`, `name`, `title`, `views`, `likes`, `comments`, `shares`, `thumbnail`, `url`, `id`, `createdAt`를 가진다.
- region은 `TIKTOK_REGION = "KR"`이다.
- 캐시 key는 `("tiktok", tuple(accounts))`이다.

## Key Function Signatures

아래 함수명은 실제 소스의 공개 함수와 내부 헬퍼를 기준으로 한다.
테스트 헬퍼는 동작 계약을 보여주는 경우에만 포함한다.

### `register()`

- `tiktok` source를 `tiktok_accounts.json`에 연결한다.

### `_tiktok_item(v)`

- tikwm video dict를 프론트 카드 shape로 변환한다.
- cover가 없으면 origin_cover를 fallback으로 사용한다.

### `fetch_tiktok_user(handle: str)`

- `/user/posts` endpoint를 호출하고 videos를 `_tiktok_item`으로 매핑한다.

### `fetch_tiktok_trending()`

- `/feed/list` endpoint를 KR region으로 호출한다.

### `get_tiktok(force: bool)`

- 트렌딩 posts를 먼저 가져오고 계정별 posts를 병렬로 추가한다.
- id 기준 dedupe 후 `(posts, accounts, fetched_at)`을 반환한다.

### 테스트 함수 지도

- `TiktokToolTest.test_tiktok_item_mapping_and_cover_fallback`
- `TiktokToolTest.test_fetch_tiktok_user_parses_tikwm_response`
- `TiktokToolTest.test_fetch_tiktok_trending_parses_tikwm_response`
- `TiktokToolTest.test_tiktok_fetch_exception_fallbacks`
- `TiktokToolTest.test_get_tiktok_dedupes_and_cache_key_includes_accounts_tuple`

## Dependents

이 모듈을 import하거나 HTTP 라우트로 소비하는 쪽이다.

- `src/tiktok/__init__.py` re-exports `get_tiktok`, `register`
- `src/main.py` calls `tiktok_tool.register()` at import time
- `src/main.py` calls `tiktok_tool.get_tiktok` in `/api/tiktok`
- `src/main.py` account POST route updates `tiktok` source
- `src/frontend/index.html` calls `/api/tiktok` and `/api/tiktok/accounts`
