## Module Responsibility

`src/ai_news/`는 AI 영상 뉴스와 Hugging Face 영상 생성 모델 목록을 모은다.
국내외 Google News RSS를 병렬로 읽고, Hugging Face 모델 API를 pipeline과 정렬 기준별로 호출한다.
추가로 TikTok과 YouTube URL의 oEmbed metadata를 가져오는 작은 helper를 제공한다.

### 책임 경계

- NEWS_FEEDS query 구성을 소유한다.
- RSS item 파싱과 timestamp 정렬을 소유한다.
- Hugging Face pipeline별 latest/trending split을 소유한다.
- oEmbed 지원 host routing을 소유한다.

### 운영 관점

- AI data는 `{news, models, fetchedAt}` 형태로 main.py에서 응답된다.
- models는 `latest`와 `trending` 배열을 가진다.
- model item은 `id`, `likes`, `downloads`, `pipeline`, `createdAt`를 가진다.
- news item은 `region`, `title`, `source`, `link`, `ts`를 가진다.

## Key Function Signatures

아래 함수명은 실제 소스의 공개 함수와 내부 헬퍼를 기준으로 한다.
테스트 헬퍼는 동작 계약을 보여주는 경우에만 포함한다.

### `fetch_news()`

- NEWS_FEEDS를 최대 2 worker로 fetch한다.
- RSS item의 title, source, link, pubDate timestamp를 만든다.
- 최신순으로 최대 40개를 반환한다.

### `fetch_hf_models()`

- text-to-video와 image-to-video pipeline을 createdAt/trendingScore 정렬로 호출한다.
- 중복 model id를 제거하고 latest/trending을 나눠 반환한다.

### `get_ai_data(force: bool)`

- news와 models fetch를 병렬 실행한다.
- 캐시 key는 `("ai",)`이다.

### `fetch_oembed(url: str)`

- tiktok.com은 TikTok oEmbed endpoint로 라우팅한다.
- youtube.com 또는 youtu.be는 YouTube oEmbed endpoint로 라우팅한다.
- 미지원 host는 `{ok: False, reason: "unsupported"}`를 반환한다.
