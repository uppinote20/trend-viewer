## Module Responsibility

`src/reels/`는 등록 계정의 Instagram Reels를 조회수 중심으로 모은다.
Instagram web profile info endpoint를 호출하고 timeline media 중 영상만 카드 데이터로 변환한다.
계정 목록은 shared accounts registry에 `reels` source로 등록된다.

### 책임 경계

- Instagram app id 상수를 보존한다.
- 계정별 병렬 fetch를 소유한다.
- 영상-only 필터와 caption 첫 줄 title 생성을 소유한다.
- 결과는 views 내림차순으로 정렬한다.

### 운영 관점

- Reel item은 `account`, `title`, `views`, `likes`, `comments`, `thumbnail`, `url`, `takenAt`를 가진다.
- 계정 목록은 `config/reels_accounts.json`에 저장된다.
- main.py는 `/api/reels`에서 `reels[:80]`으로 응답을 제한한다.

## Key Function Signatures

아래 함수명은 실제 소스의 공개 함수와 내부 헬퍼를 기준으로 한다.
테스트 헬퍼는 동작 계약을 보여주는 경우에만 포함한다.

### `register()`

- `accounts_tool.register_source("reels", "reels_accounts.json", DEFAULT_IG_ACCOUNTS)`를 호출한다.

### `fetch_ig_reels(username: str)`

- Instagram web profile info API를 호출한다.
- `is_video`가 아닌 media는 건너뛴다.
- reel item dict를 반환한다.

### `get_reels(force: bool)`

- 등록 계정을 읽고 병렬 fetch를 실행한다.
- 캐시 key는 `("reels", tuple(accounts))`이다.
- 반환값은 `(reels, accounts, fetched_at)`이다.

## Dependents

이 모듈을 import하거나 HTTP 라우트로 소비하는 쪽이다.

- `src/reels/__init__.py` re-exports `get_reels`, `register`
- `src/main.py` calls `reels_tool.register()` at import time
