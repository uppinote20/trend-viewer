## Module Responsibility

`src/x_twitter/`는 X 공개 syndication profile HTML에서 최신 포스트를 뽑는다.
`__NEXT_DATA__` script JSON을 찾아 timeline entries를 재귀 탐색하고, 좋아요·댓글·리트윗·조회수 통계를 정규화한다.
X 계정명은 대소문자 보존이 필요하므로 accounts registry에 preserve_case로 등록한다.

### 책임 경계

- syndication URL 조립을 소유한다.
- timeline entries 재귀 탐색을 소유한다.
- tweetResult fallback을 소유한다.
- 포스트 응답 shape를 프론트 소셜 카드에 맞춘다.

### 운영 관점

- Post item은 `account`, `name`, `text`, `likes`, `replies`, `retweets`, `views`, `media`, `url`, `createdAt`를 가진다.
- X source는 계정 대소문자를 보존한다.
- 프론트 정렬 메뉴는 likes, replies, retweets, views를 사용한다.

## Key Function Signatures

아래 함수명은 실제 소스의 공개 함수와 내부 헬퍼를 기준으로 한다.
테스트 헬퍼는 동작 계약을 보여주는 경우에만 포함한다.

### `register()`

- `x` source를 `x_accounts.json`과 기본 계정 목록에 연결한다.
- `preserve_case=True`를 사용한다.

### `_find_timeline_entries(node)`

- 중첩 JSON에서 `timeline.entries` 리스트를 찾아 반환한다.
- 없으면 `None`을 반환한다.

### `fetch_x_posts(username: str)`

- syndication HTML을 받고 `__NEXT_DATA__` JSON을 파싱한다.
- tweet 또는 tweetResult.result를 포스트 dict로 변환한다.

### `get_x_posts(force: bool)`

- 계정 목록을 읽고 최대 3 worker로 계정별 fetch를 실행한다.
- 캐시 key는 `("x", tuple(accounts))`이다.
- 반환값은 `(posts, accounts, fetched_at)`이다.

### 테스트 함수 지도

- `XTwitterToolTest.test_register_preserves_case`
- `XTwitterToolTest.test_find_timeline_entries_nested`
- `XTwitterToolTest.test_fetch_x_posts_parses_next_data_html`

## Dependents

이 모듈을 import하거나 HTTP 라우트로 소비하는 쪽이다.

- `src/x_twitter/__init__.py` re-exports `get_x_posts`, `register`
- `src/main.py` calls `x_twitter_tool.register()` at import time
- `src/main.py` calls `x_twitter_tool.get_x_posts` in `/api/x`
- `src/main.py` account POST route updates `x` source
- `src/frontend/index.html` calls `/api/x` and `/api/x/accounts`
